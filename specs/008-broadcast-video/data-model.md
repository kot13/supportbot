# Data Model: 008-broadcast-video

## Существующие сущности

### `broadcast_messages`

Без обязательных новых полей: текст сообщения и формат остаются источником подписи к медиа.

### `broadcast_attachments`

| Поле | Использование для видео |
|------|---------------------------|
| `broadcast_message_id` | FK на запись рассылки |
| `ordinal` | Порядок в альбоме (0..n-1) |
| `original_filename` | Имя файла от клиента |
| `mime_type` | Например `video/mp4`, `video/quicktime` |
| `size_bytes` | Размер загрузки |
| `telegram_file_id` | После успешной отправки (если цепочка обновляет поле — сохранить поведение) |

**Новая таблица не требуется** для MVP.

### Опциональное расширение (не MVP)

- Колонка **`attachment_kind`**: `'image' | 'video'` — упрощает отчёты и валидацию на уровне БД.
- Либо CHECK по `mime_type like 'video/%'`.

## Входные данные (концептуально)

- **Видео-вложение**: массив байтов, MIME, исходное имя, порядок.
- **Взаимоисключение**: в один момент времени форма хранит либо набор изображений, либо набор видео (клиентское состояние).

## Состояния доставки

Без изменения модели: те же `delivery_results` / статусы, что для текущей рассылки.
