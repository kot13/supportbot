# Contract: POST `/api/broadcasts` (расширение для видео)

Существующий endpoint сохраняет **cookie-сессию** и **`requireAuth`**.

## JSON-тело (без вложений)

Без изменений: текст-only рассылка без файлов.

## `multipart/form-data`

Новые/уточнённые поля (в дополнение к текущим `content`, `format`, `targetMode`, `chatIds`, `images`):

| Поле | Описание |
|------|-----------|
| `videos` | Повторяющееся поле файлов (`videos` × N), каждый — видеофайл, прошедший серверную валидацию |

### Правила v1 (план реализации)

- Одновременно в одном запросе: **либо** все непустые `images`, **либо** все непустые `videos`, но **не оба набора**. Иначе ответ **400** с кодом валидации.
- Пустой текст `content` — по-прежнему отклоняется (как текущая логика с обязательным сообщением).
- Длина подписи: при наличии любого медиа (изображения или видео) действует лимит **1024** символов на подпись (согласовано с текущим правилом для изображений).

### Ответ

Форма успеха/ошибки совместима с текущей: `{ ok: true, data: { id, summary } }` / `{ ok: false, error: { message, code } }`.

### Ошибки (примеры кодов)

- `VALIDATION` — неверный MIME, превышен размер, слишком много файлов, смешение images+videos, неверные `chatIds`.
