# Quickstart: проверка видео в рассылке (008)

## Предусловия

- Приложение запущено, БД мигрирована, авторизация админа работает.
- У бота настроен токен; есть тестовые чаты.
- Подготовьте **короткий** файл MP4 (несколько МБ), укладывающийся в лимит размера, выбранный в коде.

## Ручная проверка (MVP)

1. Откройте `/broadcast`, введите текст подписи (учитывайте лимит 1024 символа при медиа).
2. Прикрепите одно видео (через новый контрол выбора видео). Убедитесь, что ранее выбранные изображения сброшены, если они были.
3. Откройте подтверждение: в сводке должно быть видно количество видео.
4. Подтвердите отправку; проверьте Telegram-чат получателя: видео и подпись.
5. Повторите с **двумя** короткими видео (если включено в реализации): альбом из видео и одна подпись на первом сообщении группы.
6. Негативные кейсы: файл > лимита, не-video MIME — ошибка в UI до отправки всем.

## Автоматические тесты

- Unit: функции валидации MIME/размера/количества.
- При наличии стабов Telegram: integration для `POST /api/broadcasts` с multipart.
