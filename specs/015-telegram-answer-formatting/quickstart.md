## 3. Ручная проверка в Telegram

### 3.1 Ответ со структурой (без заголовков Markdown)

**Вопрос**: «Как подключить SDK на Android?»

**Ожидание**:

- [ ] Нет строк с `#`, `##`, `###`
- [ ] Шаги оформлены списком (`1.` или `-`)
- [ ] Подзаголовки выделены **жирным**
- [ ] Есть кликабельная ссылка на документацию

### 3.2 Ответ с кодом

**Вопрос**: «Покажи пример инициализации SDK на Android»

**Ожидание**:

- [ ] Инлайн-код (`apiKey`) моноширинный
- [ ] Блок кода с переносами строк читаем
- [ ] Нет сырого ` ``` ` в сообщении

### 3.3 Ответ, провоцирующий таблицу

**Вопрос**: «Какие параметры у метода создания сторис и что они означают?»

**Ожидание**:

- [ ] Нет символов `|` в табличной раскладке
- [ ] Параметры перечислены списком или строками `имя — описание`

### 3.4 Длинный ответ

**Вопрос**: широкий вопрос по интеграции SDK (ответ > 4096 символов при необходимости симулировать)

**Ожидание**:

- [ ] Сообщение приходит несколькими частями
- [ ] Ни одна часть не обрывается посередине блока кода
- [ ] Форматирование в каждой части корректно

### 3.5 История в панели

1. Отправить тестовый вопрос боту
2. Открыть `/chats` → соответствующий чат

**Ожидание**:

- [ ] Текст ответа бота без `#`, `|---|`, сырой таблицы
- [ ] Ссылки и структура читаемы в `whitespace-pre-wrap`
