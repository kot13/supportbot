# Quickstart: проверка подтверждения рассылки (007)

## Предусловия

- Приложение запущено (`npm run dev` или `npm start`), БД мигрирована, есть тестовый вход админа (см. `.env`: `ADMIN_LOGIN` / `ADMIN_PASSWORD`).
- В системе есть хотя бы один чат (бот добавлен в чат), если проверяете режим «все» или выбор подмножества.

## Ручная проверка

1. Войти в панель → открыть `/broadcast`.
2. Ввести короткий текст, при необходимости выбрать режим получателей и вложения.
3. Нажать действие, которое инициирует отправку (после реализации — оно открывает окно подтверждения, а **не** шлёт запрос сразу).
4. В окне проверить сводку: охват, число изображений, фрагмент текста.
5. **Отмена** — окно закрывается, текст и настройки на месте, в истории рассылок новой записи нет.
6. Снова открыть подтверждение → **Подтвердить** — поведение как у текущей рассылки (успех/ошибка, счётчики).

## Автоматическая проверка

- После добавления сценария в Playwright: `npx playwright test` (или целевой файл), с учётом переменных окружения для логина.
