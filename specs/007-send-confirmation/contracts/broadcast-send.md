# Contract: Broadcast send (unchanged server API)

Фича **007** не меняет контракт `POST /api/broadcasts`. Клиент по-прежнему отправляет:

- **`application/json`** (без изображений): `content`, `format`, `targetMode`, `chatIds` при `targetMode === "subset"`.
- **`multipart/form-data`** (с изображениями): те же поля плюс файлы `images`.

Авторизация: сессия через cookie; неавторизованный запрос — как сейчас (отказ).

## Изменение только в клиентском потоке

| До 007 | После 007 |
|--------|-------------|
| Один клик по «Send broadcast» → сразу `POST` | Первый клик открывает UI подтверждения; `POST` только после явного подтверждения во втором шаге |

Ответы успеха/ошибки, коды статусов и форма JSON-тела остаются прежними. Дублирующий серверный флаг «подтверждено» **не** вводится: защита от случайной отправки — ответственность UI.
