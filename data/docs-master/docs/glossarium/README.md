# Glossarium

This is the glossarium for frequently asked questions by users of InAppStory Console, like project managers, designers and analysts.

Here you can find answers for questions such as "How to target stories?", "How to personalize user experience with stories?", "What does "impression" in statistics mean?" and so on.

### Workflow

The process of InAppStory implementation takes a quantity of steps that your team will have to go through.

1. Creating your first stories and getting to know the admin console;
2. Implementing basic IAS features like feeds and [**onboardings**](./terms/onboardings.md);
3. Learning about additional features like [**targeting**](./targeting/targeting.md), [**personalization**](./personalization/placeholder-personalization.md) and [**reactions**](./terms/reactions.md);
4. Learning to work with InAppStory [**statistics**](./statistics/terms.md);
5. Configuring the [**appearance**](./appearance/how-to-change-design.md) of the InAppStory widget to fit to your needs.
